Compatibility

Versioning Policy

Versioning stays predictable, with clear migration guidance whenever behavior changes.

  • SemVer across core and SDK.
  • Deprecated contracts remain for at least one release cycle unless safety requires immediate removal.
  • Behavior changes are validated before release through repeatable checks.
Security

Data Handling and Safety

Security is default-first: sensitive data, scoped access, and clean operational hygiene.

  • No secrets in source control; rotate on incident or role change.
  • Minimize and redact sensitive payloads before storage when possible.
  • Use TLS transports and least-privilege DB access.
Deployment

Runtime and Release Readiness

Deploy with clear environment profiles and strict release gates before production rollout.

  • Local, CI, staging, and production runtime matrix is explicit.
  • Workspace compile/tests and integration parity are release gates.
  • Scope and provider settings stay explicit per environment.
